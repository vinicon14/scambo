# i18n: Scambo em 20 Idiomas

## Objetivo
Traduzir a plataforma Scambo para 20 idiomas para maximizar alcance orgânico e permitir participação internacional.

## Stack
- **next-intl** v3+ — roteamento, traduções, formatação locale-aware
- **Middleware Next.js** — detecção de locale (Accept-Language) + redirecionamento + cookie de preferência
- **JSON** plano para mensagens — um arquivo por idioma em `src/messages/`

```
    api/                   -- fora de [locale], sem tradução nos endpoints

```

## Locales

20 idiomas: `pt` (default), `es`, `en`, `fr`, `de`, `it`, `ja`, `ko`, `zh`, `ru`, `ar`, `hi`, `nl`, `pl`, `sv`, `da`, `fi`, `no`, `tr`, `ro`.

## Seletor de Idioma

No `Header.tsx`: `<select>` com bandeiras/abreviações dos 20 idiomas. Usa `useLocale()` e `usePathname()` para navegar entre variantes mantendo a rota atual.

## SEO — hreflang

O layout em `[locale]/layout.tsx` gera tags `<link rel="alternate" hreflang="pt" href="...">` para todas as 20 variantes via `generateStaticParams` + `generateMetadata`. Essencial para evitar conteúdo duplicado.

## Migração

1. Instalar `next-intl`
2. Criar `src/i18n/routing.ts` com locales + pathnames
3. Criar `src/i18n/request.ts` com mensagens async
4. Criar `middleware.ts` com locale detection
5. Criar `src/messages/pt.json` com todas as strings extraídas
6. Mover `app/(...)/page.tsx` → `app/[locale]/(...)/page.tsx`
7. Atualizar `layout.tsx` para usar `NextIntlClientProvider` + hreflang
8. Gerar `src/messages/es.json`, `en.json`, ... via IA
9. Adicionar seletor de idioma no Header
10. Traduzir JSON-LD (FAQPage, HowTo, WebSite)
11. Traduzir manifest.json e robots.txt
12. Testar rotas, redirecionamentos, formatação BRL/...
13. Configurar Vercel para deploy com todos os locales

## Não incluído (YAGNI)
- Tradução do painel admin — será mantido em pt-BR por ser interno
- Tradução dos slugs de admin — /admin permanece /admin em todos idiomas
- CMS de traduções — JSON files + git é suficiente para o escopo
